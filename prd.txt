## 1. Executive Summary & Vision

**AgentGuard** is the **runtime security layer for autonomous AI agents** — not just another secret scanner, but a complete secure-memory infrastructure that enforces the **Checkpoint → Scan → Taint → Vault → Replace → Continue** lifecycle at every critical boundary.

It protects long-lived agents, shared memory, vector stores, tool outputs, and multi-agent communication from credential leakage while enabling zero-knowledge execution and full compliance.

**Vision:** Become “the Cloudflare of AI agents” — the default middleware developers install because operating without it feels irresponsible. Self-hosted is completely free; hosted uses x402 agent-native micropayments.

This PRD merges the **existing v1 SDK architecture** (as documented in the official docs) with all high-impact improvements from both expert reviews. It defines a complete, production-ready v2.0 that ships the P0 must-haves and sets up P1/P2 for enterprise dominance.

## 2. Problem Statement & Opportunity

### Core Problem
Autonomous agents leak secrets into:
- Conversation history / memory
- Tool outputs & summaries
- Vector store embeddings
- Agent-to-agent messages
- Checkpoints & persistent state

Once leaked, regex scanners fail (summaries, rewrites, embeddings no longer match patterns). Traditional secret managers don’t operate *inside* the agent runtime.

### Market Opportunity
- Explosion of long-lived, multi-agent systems (LangChain, CrewAI, AutoGen, Anthropic, OpenAI agents)
- Regulatory pressure (SOC2, GDPR, HIPAA for AI)
- Agent economy (on-chain agents paying for services via x402)
- Vector DBs becoming permanent memory

AgentGuard solves the full lifecycle with **automated remediation + taint tracking + boundary enforcement**.

## 6. High-Level Architecture

**Monorepo remains unchanged**  
`packages/@agentguard/{core, vault, patterns, adapters, server, cli, x402}` + meta `agentguard`

**Key Data Flow (enhanced)**
1. Agent state → Checkpoint trigger
2. Scanner (incremental + taint-aware) → detects + assigns `secret_id`
3. Taint Propagation Engine tags derived content
4. Pruner (in Ephemeral Worker) → vault.write + reference replacement
5. Vault (with ACLs + crypto refs + TTL) → stores secret
6. Clean state + vaultRef returned
7. Audit log (tamper-evident hash chain)

**Ephemeral Core** now uses bounded worker pool + stronger isolation (`isolated-vm` / `vm2` fallback).

**New Modules**
- `TaintEngine` (internal to core)
- `VectorStoreAdapters` (new package or extension of adapters)
- `PromptShield` (new detection layer)
- Enhanced `Vault` with ACL + crypto + TTL + rotation hooks

## 7. Functional Requirements (Existing + Improvements)

### 7.1 Core API (enhanced v1)
- `new AgentGuard(config)` – unchanged + new options:
  - `taintEnabled: boolean`
  - `vectorStores: []`
  - `promptShield: { enabled, rules }`
  - `workerPool: { size, ttlJobs }`
- `guard.wrap(fn, state)` – now supports taint propagation and streaming mode
- `guard.checkpoint(state)` – now incremental by default
- `guard.scan()` / `guard.prune()` – unchanged + taint support
- **New:** `guard.stream(tokens)` for incremental token-by-token scanning

### 7.2 Scanner + Taint Tracking (P0)
- Recursive deep scan of objects/arrays/strings/conversation history
- 40+ built-in patterns (existing) + live-updatable registry (GitHub/S3)
- **Taint Propagation Engine**:
  - On detection → assign immutable `secretId`
  - Tag all derived values (summaries, tool outputs, embeddings, rewrites) with `sourceSecretIds: []`
  - Context-aware scoring (location + entropy)
- Pattern Trie / Aho-Corasick for initial fast pass

### 7.3 Pruner + Vault (P0)
- Existing strategies (REDACT, REMOVE, REPLACE) + new `VAULT_WITH_TAINT`
- **Cryptographic Vault References**:
  - Format: `vault:v1:<randomId>:<hmac>:<expiry>`
  - Never includes secret type
  - HMAC-signed, short-lived (configurable TTL)
- **Secret ACLs / Multi-Agent Boundary Enforcement**:
  - `vault.write(secret, { visibleTo: ["agent:treasury", "wallet:0x…"] })`
  - Resolution fails for unauthorized agents
- **Secret Rotation Hooks**:
  - `onSecretFound(match, rotationHook?)` → auto-rotate AWS/HashiCorp/etc. + update ref

### 7.4 Vector Store Sanitization (P0)
- New adapters: Chroma, Pinecone, Weaviate, Qdrant, LanceDB, Milvus (and extensible)
- `guard.wrapVectorStore(store)` – intercepts `addDocuments`/`upsert` etc.
- Scan → taint → prune → vault → embed clean vectors

### 7.5 Ephemeral Core (hardened)
- Bounded worker pool (pre-warm, recycle after N jobs, forced kill on timeout)
- Stronger isolation (`isolated-vm` / `vm2` inside worker where possible)
- Explicit buffer zeroing + memory scrub
- Best-effort guarantee clearly documented

### 7.6 Checkpoint System (enhanced)
- All existing triggers + new `VECTOR_WRITE`, `BEFORE_EMBED`
- Custom triggers can now `intercept: true` for taint/vector flows

### 7.7 Framework Adapters (expanded)
- Existing (LangChain, Anthropic, OpenAI, Express)
- New vector store wrappers
- Python equivalents for all major frameworks

### 7.8 Audit & Compliance (P1)
- Existing structured logs
- **Tamper-evident hash chain** (`hash(prev + current)`)
- Optional blockchain anchoring (Base)
- Signed Ed25519 option
- Never logs raw/redacted critical secrets (outcome-only)

### 7.9 PromptShield (P1)
- New detection layer for:
  - Prompt injection / jailbreaks
  - Tool abuse
  - Data exfiltration attempts
  - “Ignore previous instructions” patterns
- Runs in parallel or pre-checkpoint
- Configurable severity + block actions

### 7.10 x402 Payments (unchanged + documented)
- Fully autonomous agent payments via on-chain identity
- Self-hosted = free forever

### 7.11 CLI & Server
- `agentguard scan --git`, pre-commit hooks, GitHub Actions
- Enhanced server middleware with x402 + new features

### 7.12 Patterns & Extensibility
- Built-in 40+ patterns (critical → medium severity)
- Custom pattern registry + live updates
- Entropy + optional ML anomaly detection (P2)

## 8. Non-Functional Requirements

### Performance
- <50ms median checkpoint on 10k-token history
- Streaming mode for real-time token arrival
- Worker pool: max 10 concurrent, recycle after 100 jobs
- Benchmarks published for LangChain/Anthropic

### Security
- Zero raw secrets post-prune (best-effort + isolation)
- Taint never leaks
- Vault refs cryptographically protected + TTL
- Rate limiting + max depth/size guards
- No secret type in references or logs

### Reliability
- Graceful degradation if worker fails
- Configurable fallback strategies
- Full test coverage (>90%)

### Scalability
- Horizontal scaling via worker pool
- Multi-tenancy in server (agent-wallet isolation)

### Observability
- OpenTelemetry traces
- Prometheus metrics (`secrets_found_total`, `prune_duration_ms`, etc.)
- Slack/PagerDuty/webhook alerts
